## Key Patterns

- `packages/editor/src/features/link.ts` — `FeatureSpec` shape: `marks`, `parserTokens`, `inline.{priority, markNames, scan, extRanges}`. `scan` uses `markConsumed`, walks `consumed` Uint8Array, emits `InlineSpan` with `openFrom/openTo/contentFrom-contentTo/closeFrom/closeTo` + optional `delimRanges` and `attrs`. Link's empty-label `delimRanges` show forceVisible delims when content is empty — image's empty-url should return `[]` (no delim → source always visible).
- `packages/editor/src/features/inline-decorations.ts` — `syntaxHintsPlugin` reads `getDelims(state)` (which reads `normalizeInlineKey.getState`). Each delim becomes a `Decoration.inline`. Cursor inside `[spanFrom, spanTo]` → `syntax-hint` (visible pending), outside → `syntax-hidden` (font-size:0). Default (no `delimRanges`) emits only `[openFrom,openTo]` and `[closeFrom,closeTo]`; image uses entire `[openFrom,closeTo]` as ONE delim range.
- `packages/editor/src/features/inline-normalize.ts` — `normalizeInlineKey` PluginKey exports `NormalizeInlineState = { blocks: [{blockPos, blockStart, spans}], delims: DelimRange[] }`. Image-widgets plugin reads `plan.blocks[*].spans` directly, computes widget pos = `block.blockStart + span.closeTo`.
- `packages/editor/src/specs/cursor-render.ts` — exact `Decoration.widget(pos, factory, { side })` shape to mirror. Returns DecorationSet via `props.decorations`.
- `packages/editor/src/specs/features/live-link.cases.ts` — case schema: `id`, `title`, `initialMarkdown`, `keyevents[]`, `checkpoints[{step, expectedProjection, expectedMarkdown}]`. Uses `<pending>...</pending>` for syntax-hint, raw text for hidden state (because DEFAULT_TAGS handles A/SUB/SUP etc as wrappers). Cursor is `|`.
- `packages/editor/src/specs/runner.ts:192-200` — `A:` serializer pattern for IMG: read attrs via `el.getAttribute(...)`, escape with `escapeAttribute()`, join nullable attrs by conditional concat. Image must follow same style: `<img src="..." alt="..." title="...">` with title/alt omitted when null/empty.
- `packages/editor/src/features/inline-parse.ts:139-149` — `parseInline` iterates features in priority order (ascending — lowest priority runs first). Crossing-rejection via `crosses()`. Image must guarantee it runs before link (priority=3) AND not lose to emphasis (priority=2, ties broken by featureSpecs order — image must precede emphasis OR use priority<2).
- `packages/editor/src/markdown/parser.ts:208-238` — `tokenSpecs` table → handlers; feature `parserTokens` registered last via `collectParserTokens()`. Markdown-it `image` is a single self-closing token (`tok.type === 'image'`, attrs `[['src', href], ['alt', '']]`, optional `['title', t]`, `tok.content === alt text`). Handler signature: `(state, token, schema) => void`.

## Conventions

- **Naming**: feature file = `<name>.ts` (lowercase). Spec file = `live-<name>.cases.ts`. Plugin export from feature = function returning `Plugin<DecorationSet>` (camelCase + `Plugin` suffix, e.g. `imageWidgetsPlugin`). CSS class = `.md-<purpose>-*` (e.g. `.md-image-widget`, `.md-image-source`). Spec id = `live-<name>`. Case id = `live-<name>-<behavior>`.
- **Imports**: relative `.ts` extensions throughout (`./inline-parse.ts`). Type-only imports use `import type`. `FeatureSpec` from `./_types.ts`, `InlineFeatureSpec`/`InlineSpan`/`markConsumed` from `./inline-parse.ts`. ProseMirror imports from individual `prosemirror-*` packages.
- **Error handling**: scan returns empty array on no-match; defensive `?? ""` / `?? null` for attrs. `markConsumed` mutates Uint8Array in place. Widget plugin: guard `if (!plan) return DecorationSet.empty;` and `if (!src) continue;`. No try/catch needed.
- **Types**: `MarkSpec` cast via `as MarkSpec`. `Plugin<DecorationSet>` for decoration plugins. `EditorState` from `prosemirror-state`, `Decoration`/`DecorationSet` from `prosemirror-view`.
- **Testing**: One case per behavioral pivot. Use `<pending>...</pending>` wrapper for syntax-hint visible delim. Use `|` for cursor. Use `{img}` literal as `<img src="y" alt="x">` (spec's notation; real case uses literal HTML). `initialMarkdown` must NOT contain `|`. `expectedMarkdown` round-trip required for every checkpoint.

## Risks

- **Widget position vs side**: Spec example projection `<p>{img}<pending>![x](y)</pending>|</p>` puts `{img}` BEFORE the source. But `Decoration.widget(closeTo, ..., { side: 1 })` places widget AFTER the source in DOM order. Builder needs to use `Decoration.widget(openFrom, ..., { side: -1 })` to render BEFORE source, OR re-read spec's intent. Failure mode: 8/8 cases fail projection assertion on first run. Mitigation: builder writes TDD case 1 first, observes DOM order, picks correct pos+side combo.
- **Priority tie with emphasis (both = 2)**: `featureSpecs` order is the tie-breaker (stable sort). Builder must place `image` BEFORE `emphasis` in the array (spec recommends "above link" — sufficient since link is currently first). If misplaced, `**![x](y)**` could give strong precedence over image consumption.
- **CSS location ambiguity**: Spec lists `apps/playground/src/styles.css` (doesn't exist; actual file is `style.css`). Editor package has its own `packages/editor/src/style.css`. Recommend editor's file since the widget is editor-wide. Open item in spec acknowledges this.
- **Spec case 7 (`![y](z)` does-not-trigger-on-bracket-only)** — title is misleading. The case keyevents are `[, x, ], (, y, )` → input `[x](y)` which should produce link only. Builder must NOT confuse with `![x](y)` testing. The actual image-trigger validation is implicit in case 1.
- **`tok.content` vs `tok.attrGet('alt')`**: markdown-it sets `attrs[1] = ['alt', '']` always empty; the alt text lives in `tok.content`. Spec's snippet `const alt = token.content ?? "";` is correct — builder must NOT use `attrGet('alt')`.
- **Alt with inline markdown** (`![**a**](b)`): markdown-it parses inner alt into `tok.children`. Spec correctly ignores `children` and uses `tok.content` (raw text). This already gives plain-text alt — no extra suppression needed at parse time. But `parseInline` will re-scan the doc text `![**a**](b)` and image's scan will consume the whole segment via `markConsumed(openFrom, closeTo)` — so emphasis can't claim `**a**` inside it (provided image runs first).
- **`extRanges` is currently dead code**: No consumer reads it. Still required for forward-compat and to satisfy the established `InlineFeatureSpec` shape used by every other inline feature. Image should provide it mirroring link's pattern with `!` prefix requirement.
- **`Decoration.widget` and `pos` resolution**: Widget pos uses doc coordinates (`block.blockStart + span.closeTo`). Must NOT use `block.blockPos` (which points at the textblock open token, off by one).
