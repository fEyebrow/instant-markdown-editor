Context · 01

The problem

01编辑器目前只能通过手输 Markdown 文本 ![alt](url) 插入图片，URL 必须已存在。用户截图后粘贴、或把本地图片文件拖进编辑器时没有任何处理——文件被浏览器默认行为吞掉或直接丢弃，本地图片无法进入文档。

02类 Typora 的 WYSIWYG 体验要求：粘贴/拖拽图片 → 自动上传到用户自己的云存储 → 拿到 URL 后以 ![alt](url) 进入文档。编辑器是无后端的库，上传能力必须由宿主应用注入。

Commit · 02

Goals

1 粘贴剪贴板中的图片文件（截图、复制的本地文件）自动上传并在光标处插入 ![alt](url)
2 拖拽本地图片文件到编辑器，上传后插入到拖放坐标对应位置
3 编辑器保持后端无关：EditorOptions 新增 uploadImage?: (file: File) => Promise<string> 回调
4 上传期间显示占位 widget（本地 blob 预览 + 上传中样式），位置随并发编辑正确映射
5 上传失败时占位移除、文档零残留，并通过 onImageUploadError 回调通知宿主
Signal · 03

Done when…

9 signals
  • 01 提供 uploadImage 时：粘贴 image/* 文件 → 占位出现 → Promise resolve 后该位置出现 ![alt](url) 文本，图片 widget 正常渲染
  • 02 上传期间 getMarkdown() 输出不包含未完成的图片
  • 03 上传期间在占位前插入/删除文本，最终 ![alt](url) 插入位置仍正确
  • 04 Promise reject 后：占位消失，markdown 与上传前一致，onImageUploadError(file, error) 被调用
  • 05 未提供 uploadImage 时：编辑器不拦截 paste/drop，行为与现状一致
  • 06 一次粘贴/拖拽多个图片文件：按顺序全部上传并插入
  • 07 非 image/* 文件不处理，走默认行为
  • 08 alt 默认为文件名去扩展名
  • 09 pnpm ready（check + test + build）全量通过

MVP — must ship ×9
  • EditorOptions 新增 uploadImage / onImageUploadError 回调— 库的对外契约，一切功能的入口
  • image-upload 插件：handlePaste 拦截剪贴板 image/* 文件— 核心场景：截图粘贴
  • image-upload 插件：handleDrop 拦截拖入的 image/* 文件，posAtCoords 定位— 核心场景：拖拽本地文件
  • 上传占位 widget decoration：本地 blob 预览 + 上传中样式，位置随事务映射— 已确认的上传中 UX
  • 成功后在占位位置插入 ![alt](url) 文本，复用现有 image feature 渲染— 不引入新 schema node，架构最小侵入
  • 失败移除占位 + onImageUploadError 回调— 已确认的失败处理
  • 多文件按顺序上传插入— 粘贴/拖拽天然可能带多个文件
  • playground demo uploadImage（模拟延迟 + 本地 URL）— 手动验证入口
  • jsdom 测试：mock File + uploadImage 驱动处理函数，断言 projection 与 markdown— spec runner 不支持 paste/drop 事件
Full — target outcome ×2
  • 拖拽时的插入位置指示（prosemirror-dropcursor）— 拖放体验补全，独立小集成
  • 粘贴同时含文本与图片文件时文件优先的边界处理— 截图工具常见行为，避免双重插入
Stretch — if time permits ×1
  • 失败占位显示错误态 + 重试按钮
Out of scope — said no on purpose
  • 编辑器内置具体云服务 SDK（七牛/OSS/S3） — 库保持后端无关，宿主注入
  • 从其它网页拖拽图片的远程转存 — 涉及 CORS，复杂且不可靠，用户已确认排除
  • 粘贴 HTML 内嵌 base64 图片的解析转存 — 用户已确认排除
  • 上传进度条 / 取消上传 — uploadImage 契约只是 Promise<string>，进度需要扩展契约，后续再说
Future — someday, maybe
  • 图片插入后的尺寸调整、对齐等编辑能力
  • 上传进度与取消（契约扩展为带 onProgress / AbortSignal）
