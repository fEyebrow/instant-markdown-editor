Implementation Notes — Phase 1

Component 4 — Avoid adding prosemirror-model as a playground dep

Context
Cursor mapping helper needs the Node type, but prosemirror-model is only a transitive dep of the playground (it lives under packages/editor/node_modules).
Decision
Derive the doc type from the exported EditorHandle via EditorHandle["view"]["state"]["doc"] alias, so no new direct dep is needed.
Alternative
Add prosemirror-model to apps/playground/package.json. Rejected because the spec scope is "playground only" and dep changes warrant a separate decision.

Component 3 — Single-attach pattern for the window keydown listener

Context
Spec says no cleanup is needed for the playground lifetime, but Vite HMR can re-invoke renderEditor on hot updates, stacking listeners.
Decision
Hold the current handler in a module-level toggleListener and remove the prior one before re-attaching, making renderEditor idempotent under HMR.
Alternative
Attach at module scope outside renderEditor. Rejected because the listener closes over the state object created inside renderEditor.

Component 2 — Scope [hidden] fallback to the editor card

Context
Spec's failure-mode mitigation suggests [hidden] { display: none } as a global fallback against CSS overrides.
Decision
Scope the rule to .editor-card > [hidden] and drop !important — narrow enough to do its job without colliding with future [hidden] usages on other pages (e.g. specs view).
Alternative
Global [hidden] { display: none !important; }. Rejected as too broad.
