Context · 01

The problem

01playground 目前是 "左 WYSIWYG + 右只读 markdown 预览" 的双栏布局：能看 markdown，但不能直接编辑源文本；同时默认文档过于简短，不足以演示 editor 目前支持的全部 markdown 语法。

02希望把 playground 改成单栏：默认 WYSIWYG 全宽；按 Cmd+/ (Mod+/) 切换到一个可编辑的 markdown 纯文本视图，再按 Cmd+/ 切回 WYSIWYG，textarea 中的修改通过 setMarkdown 同步回富文本编辑器。同时换一份能够覆盖所有 inline 与 block 语法的默认文档，让 playground 同时承担 "使用入口" 和 "功能展示" 两个角色。

Commit · 02

Goals

1 playground 单栏化：默认 WYSIWYG 全宽显示，移除现有右侧只读 markdown 预览栏。
2 Mod+/ 在 WYSIWYG ↔ 源码 textarea 之间切换，两种视图下快捷键都生效。
3 进入源码模式时用 editor.getMarkdown() 填充 textarea；切回 WYSIWYG 时用 editor.setMarkdown() 应用 textarea 的修改。
4 默认 initialMarkdown 包含：一段介绍 packages/editor 是什么；一段把全部 inline 语法（emphasis / strikethrough / sub / sup / highlight / link / autolink / inline code）连写在一起的展示文本；每个 block 语法（atx-heading / blockquote / thematic-break / ordered-list / unordered-list）各一段简单展示。
5 不在 packages/editor 暴露新接口：已有的 getMarkdown / setMarkdown 已经覆盖需求。
Signal · 03

Done when…

6 signals
  • 01 首次打开 playground，编辑器区域全宽显示新的默认文档，原右侧 markdown 预览栏不再出现。
  • 02 默认文档在 WYSIWYG 中渲染时能看到 emphasis / strikethrough / sub / sup / highlight / link / autolink / inline code 这 8 个 inline 语法的实际效果，以及 atx-heading / blockquote / thematic-break / ordered-list / unordered-list 这 5 个 block 各被展示一次。
  • 03 在 WYSIWYG focus 时按 Mod+/，WYSIWYG DOM 被隐藏 / 卸载，出现一个 textarea，内容等于 editor.getMarkdown() 的结果，textarea 自动获得焦点。
  • 04 在 textarea focus 时按 Mod+/，textarea 隐藏，WYSIWYG 重新显示，doc 反映 textarea 的最新内容（通过 editor.setMarkdown）。
  • 05 Mod+/ 在两种视图下都不会触发浏览器默认行为；在 textarea 中也能切回。
  • 06 vp check 与 vp test 不应回退（playground 改动不应影响 packages/editor 类型与已有 spec）。

MVP — must ship ×6
  • playground 单栏布局：移除右侧 markdown 预览栏，编辑器全宽。— 切换体验的前提，与当前双栏布局互斥。
  • Mod+/ 切换 WYSIWYG ↔ textarea 源码模式（playground 层 window keydown 监听）。— 核心交互；必须在 WYSIWYG 与 textarea focus 下都能触发。
  • 进入源码模式：editor.getMarkdown() → textarea.value；textarea 接管焦点。— 源码视图必须显示当前 doc 对应的 markdown。
  • 切回 WYSIWYG：editor.setMarkdown(textarea.value)，doc 应用修改。— 双向同步，避免用户在源码模式的编辑被丢弃。
  • 新默认 markdown 文档：介绍段 + 全 inline 展示段 + 每个 block 各一段。— 替代当前简短示例，让 playground 同时承担功能展示。
  • 切换时保留光标 / 滚动位置（两种视图间映射）。— 高频交互；丢光标会让 Mod+/ 体验断裂。
Out of scope — said no on purpose
  • 在 packages/editor 内实现源码模式 / 暴露新 API。 — 现有 getMarkdown / setMarkdown 已经够用；保持 packages/editor 不感知 playground UI。
  • textarea 的 markdown 语法高亮 / 集成 CodeMirror。 — playground 是手动验证用，textarea 即可，避免引入依赖。
  • 在 localStorage 中持久化当前模式或文档内容。 — playground 行为不需要跨刷新保留。
  • 为 playground 编写 spec runner 用例。 — playground 不在 specs/ 覆盖范围内，验证以 dev server 手工为主。
Future — someday, maybe
  • 顶栏增加一个可点击的 "源码模式" 按钮，作为 Mod+/ 的视觉补充。
  • 把源码模式上升为 packages/editor 的内置展示模式。
  • playground 提供多份示例文档可切换（cheatsheet / blog post / 长文档等）。
